Follow the official workflow:

• Create a GitHub issue
• Create a new git branch
• Create several tests, which should all fail
• Fix the problem
• When done, run the tests, to show it's passing
• Create a GitHub PR, merge it, then close the ticket.
• Bump the patch version (the z in x.y.z) of package.json
• Run "bun run parser && bun run browser"
• Add an entry to the CHANGELOG.md file
• Commit and push to GitHub
• Do not push to GitHub yet

==

Follow the surgical workflow:

• Create several tests, which should all fail
• Fix the problem
• When done, run the tests, to show it's passing
• Bump the version's patch number (the z in x.y.z) of package.json
• Run "bun run parser && bun run browser"
• Add an entry to the CHANGELOG.md file
• Commit and push to GitHub

==

Release a new version of Rip:

• bump the patch number (the z in x.y.z) of package.json
• bun run parser && bun run browser
• update the CHANGELOG.md file
• commit and push
• npm publish
